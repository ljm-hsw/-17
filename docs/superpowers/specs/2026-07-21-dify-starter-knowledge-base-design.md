# Dify 入门知识库设计

## 目标

为项目负责人初次学习 Dify 提供一份可直接上传的 Markdown 知识库，用于跑通“用户提问 → 知识检索 → 大模型回答”的最小闭环。

## 文件与范围

- 最终文件名：`游迹织梦-入门知识库.md`
- 文件数量：一份
- 主要语言：简体中文
- 内容来源：已经确认的项目产品与业务设计文档
- 内容范围：项目介绍、游客使用流程、卡片绑定、现场打卡、记录与进度、AI 能力边界、常见问题

## 内容边界

- 不写入尚未由团队审核的江安校区具体点位资料。
- 不包含真实卡 UID、激活码、微信身份、API Key 或其他敏感信息。
- 不宣称系统能够获得用户实时位置。
- 不把 AI 生成内容描述成真实打卡事实。
- 对尚未实现或无法确认的能力，明确说明以实际系统状态为准。

## 文档结构

文档使用清晰的 Markdown 标题组织，方便 Dify 按段落切分：

1. 项目简介
2. 系统使用流程
3. 卡片绑定说明
4. 现场打卡与游览记录
5. AI 智能导游能力与限制
6. 隐私与安全说明
7. 常见问题
8. 建议测试问题

## 验收标准

- 文件可作为普通 Markdown 文档上传到 Dify 知识库。
- 初学者无需修改内容即可进行检索测试。
- 文档中的业务规则与现有产品设计一致。
- 回答“如何绑卡”“如何打卡”“AI 能做什么”等问题时，能够检索到独立、完整的相关段落。
- 文档中没有未确认点位、真实密钥或个人数据。
