# “游迹织梦”作品设计文档资料与引用台账

> 用途：区分项目自身设计事实与外部资料结论，确保正文只转述观点并按 GB/T 7714 标引，不复制来源文字。

## 一、项目自身设计事实

以下内容来自已确认的产品与技术设计，不作为外部引用：

- 首个验证场景为四川大学江安校区；作品名称“游迹织梦”，团队名称“成都潮人”。
- 系统由 ESP32-S3 点位终端、微信小程序、Django 后端、Vue 3 Web 管理端和 Dify/云端大模型服务组成。
- RFID/NFC 手环作为实体身份媒介；一个微信账号可绑定多张卡，一张卡同一时间只绑定一个账号。
- 绑卡优先使用手机 NFC 读取 UID，失败时允许手输 UID 并校验实体卡一次性激活码。
- 打卡按 Asia/Shanghai 自然日组织游览会话；设备事件经过认证、时间窗、随机数和幂等约束后形成可信记录。
- 拍照点遵循游览前知情授权、有效打卡、小程序即时提示、用户确认、短时拍摄授权和 `event_id` 关联；不做人脸识别、连续拍摄或无感追踪。
- AI 只通过受控工具读取用户已授权的真实打卡、路线进度、点位知识和照片元数据，不直接访问数据库，不接收原始卡 UID。

## 二、竞赛与技术资料

### [1] 2026 年乐鑫科技赛道命题

- 责任者：全国大学生物联网设计竞赛组委会、乐鑫科技。
- 标题：2026 年全国大学生物联网设计竞赛命题——乐鑫科技赛道。
- 来源：https://iot.sjtu.edu.cn/ueditor/net/upload/file/20260329/6391039317778793056390288.pdf
- 可转述结论：必备项包括指定 ESP32-S3/ESP32-C5/ESP32-P4 主控、至少一种传感器数据融合、至少一个云端大模型，以及设备下行响应或非音视频感知数据上行处理。
- 正文落点：1.3、2.2、2.3、4.1、4.2、4.6。

### [2] ESP32-S3 数据手册

- 责任者：Espressif Systems。
- 标题：ESP32-S3 Series Datasheet, Version 2.2。
- 来源：https://www.espressif.com/sites/default/files/documentation/esp32-s3_datasheet_en.pdf
- 可转述结论：ESP32-S3 集成双核 Xtensa LX7、2.4 GHz Wi-Fi、Bluetooth LE、多类外设及 SHA/AES/RSA/HMAC 等硬件安全能力，适合低功耗联网感知终端。
- 正文落点：2.2、4.2、4.5。

### [3] ESP-IDF 编程指南

- 责任者：Espressif Systems。
- 标题：ESP-IDF Programming Guide - ESP32-S3。
- 来源：https://docs.espressif.com/projects/esp-idf/en/latest/esp32s3/
- 可转述结论：ESP-IDF 提供网络、外设、系统、安全与构建工具；作品虽然可用 Arduino 快速实现演示固件，但协议和模块边界参考官方框架组织。
- 正文落点：4.2、4.7。

### [4] 非接触式卡初始化与防冲突标准

- 责任者：ISO/IEC。
- 标题：ISO/IEC 14443-3:2018 Cards and security devices for personal identification — Contactless proximity objects — Part 3: Initialization and anticollision。
- 来源：https://www.iso.org/standard/73598.html
- 可转述结论：该标准规定近距离非接触对象的轮询、初始化和防冲突过程，为 RFID/NFC 近场识别链路提供标准背景。
- 正文落点：2.2、4.2。

### [5] HMAC 标准

- 责任者：National Institute of Standards and Technology。
- 标题：FIPS PUB 198-1, The Keyed-Hash Message Authentication Code (HMAC)。
- 来源：https://csrc.nist.gov/pubs/fips/198-1/final
- 可转述结论：HMAC 将共享秘密与密码散列函数结合，用于消息认证；本系统用 HMAC-SHA256 验证设备请求完整性与来源。
- 正文落点：2.2、4.5。

### [6] JSON Web Token 标准

- 责任者：Jones M, Bradley J, Sakimura N。
- 标题：RFC 7519: JSON Web Token (JWT)。
- 来源：https://www.rfc-editor.org/info/rfc7519/
- 可转述结论：JWT 是紧凑、URL 安全的声明表示方式，可经签名或消息认证码保护；令牌应减少承载隐私敏感信息并设置有效期。
- 正文落点：4.3、4.5。

### [7] HTTP 语义标准

- 责任者：Fielding R, Nottingham M, Reschke J。
- 标题：RFC 9110: HTTP Semantics。
- 来源：https://www.rfc-editor.org/info/rfc9110/
- 可转述结论：HTTP 标准定义安全、幂等和缓存等方法语义；业务上的设备事件幂等还需用事件标识和数据库唯一约束保证。
- 正文落点：3.2、4.4、4.7。

### [8] 个人信息保护法律依据

- 责任者：全国人民代表大会常务委员会。
- 标题：中华人民共和国个人信息保护法。
- 来源：https://flk.npc.gov.cn/detail?fileId=&id=ff8081817b6472a3017b656cc2040044&title=%E4%B8%AD%E5%8D%8E%E4%BA%BA%E6%B0%91%E5%85%B1%E5%92%8C%E5%9B%BD%E4%B8%AA%E4%BA%BA%E4%BF%A1%E6%81%AF%E4%BF%9D%E6%8A%A4%E6%B3%95&type=
- 可转述结论：个人信息处理应具有明确、合理目的，并采取对个人权益影响最小的方式；处理规则需要公开透明。
- 正文落点：1.4、3.2、4.5、5.3。

### [9] 智慧旅游政策依据

- 责任者：文化和旅游部办公厅、中央网信办秘书局、国家发展改革委办公厅、工业和信息化部办公厅、国家数据局综合司。
- 标题：智慧旅游创新发展行动计划。
- 文号：办资源发〔2024〕82 号。
- 来源：https://zwgk.mct.gov.cn/zfxxgkml/zykf/202405/t20240513_952825.html
- 可转述结论：行动计划强调数字化、网络化和智能化的智慧旅游发展，并鼓励智能感知等技术建设体验新空间和新场景。
- 正文落点：1.1、2.4、5.4。

### [10] Dify 知识检索与应用编排

- 责任者：Dify。
- 标题：Knowledge Retrieval / Application Orchestration Documentation。
- 来源：https://docs.dify.ai/guides/knowledge-base/retrieval
- 可转述结论：知识库检索和工作流节点可以将外部知识与模型调用组合；本系统进一步用受控后端工具把个人事实与公共点位知识分离。
- 正文落点：2.3、3.6、4.6。

### [11] 检索增强生成论文

- 责任者：Lewis P, Perez E, Piktus A, et al.
- 标题：Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks。
- 来源：https://arxiv.org/abs/2005.11401
- 可转述结论：把参数化生成模型与可检索的外部非参数知识结合，有助于生成更具体且更具事实性的内容；本系统据此采用“真实事实快照 + 点位知识检索 + 生成后校验”的受控流程。
- 正文落点：2.3、4.6。
